# Product Spec

## 1. 한 문장 정의

imunity는 암환자와 보호자가 받을 가능성이 있는 중앙부처/지자체 공공 복지 정보를 쉬운 필터링으로 탐색하는 웹서비스다.

## 2. 핵심 사용자

- 암 진단 이후 지원 제도를 찾는 환자
- 환자를 대신해 정보를 찾는 보호자
- 병원, 상담센터, 환우 커뮤니티에서 정보를 안내하는 실무자

## 3. 사용자 문제

- 복지 정보가 중앙부처/지자체로 흩어져 있다.
- 제도명과 행정 용어가 어려워 실제 대상 여부를 빠르게 판단하기 어렵다.
- 암환자에게 관련 있는 제도만 골라 보기 어렵다.
- 지역, 나이, 소득, 성별 등 조건을 한 번에 맞춰 보기 어렵다.

## 4. MVP 목표

MVP는 “내 조건으로 받을 가능성이 있는 복지 정보를 빠르게 좁혀 보는 것”에 집중한다.

필수 기능:

- 중앙부처 복지서비스 데이터 수집
- 지자체 복지서비스 데이터 수집
- 암환자 관련성 필터링
- 지역, 나이, 성별, 소득수준, 키워드 기반 필터
- 서비스명, 지원대상, 지원내용, 신청방법, 문의처, 관련링크 표시
- 데이터 최신 수집 시각 표시
- 결과가 없을 때 조건을 완화하도록 안내

## 5. MVP 비목표

초기 버전에서 하지 않는다.

- 의료 상담 또는 치료 의사결정 지원
- 사용자의 진단서, 주민등록번호, 상세 병력 수집
- 복지 신청 대행
- 지원 대상 여부의 법적 확정 판정
- 커뮤니티, 게시판, 결제, 보험 추천

## 6. 주요 화면

### 6.1 검색/필터 화면

첫 화면은 검색 가능한 실제 서비스 화면이어야 한다. 마케팅 랜딩 페이지가 아니다.

필터:

- 거주지역: 시도, 시군구
- 나이 또는 출생연도
- 성별
- 소득수준: 모름, 기초생활, 차상위, 중위소득 구간 등
- 관심 영역: 치료비, 생계, 돌봄, 교통, 심리, 고용, 장애, 가족지원
- 검색어

### 6.2 결과 목록

각 결과는 다음 정보를 우선 표시한다.

- 서비스명
- 중앙/지자체 구분
- 소관기관 또는 지역
- 지원내용 요약
- 대상 요약
- 신청방법
- 문의처
- 관련링크
- 암환자 관련성 근거

### 6.3 상세 화면

- 원문 기반 상세 설명
- AI 요약
- 신청 전 확인해야 할 조건
- 원문/공식 링크
- 마지막 수집 시각

## 7. 콘텐츠 원칙

- “받을 수 있음”처럼 확정 표현을 피하고 “대상일 수 있음”, “확인 필요”를 사용한다.
- AI 요약과 원문 출처를 구분한다.
- 행정 용어는 쉬운 말로 풀되, 원문 필드도 보존한다.
- 사용자가 민감정보를 입력하지 않아도 기본 탐색이 가능해야 한다.

## 8. 성공 기준

- 사용자는 1분 안에 본인 지역과 조건으로 결과를 좁힐 수 있다.
- 결과 카드만 보고도 “내가 더 확인할 만한 제도인지” 판단할 수 있다.
- 각 결과에는 공식 문의처 또는 링크가 있다.
- 데이터 수집 실패 시에도 기존 캐시 데이터로 서비스가 중단되지 않는다.
